# Описание моей работы в проекте
В рамках командного проекта я разрабатывала два основных компонента: **страницу регистрации** и **игру Word Chain**, а также ряд сопутствующих вещей.
 
## Страница регистрации
- Ветка: `feat/FRONTEND_UserRegistrationPage_createRegistrationPage`
- Реализация формы регистрации с валидацией
- Проверка повторного пароля с использованием password pattern
- Интеграция с UserStore
- Подключение EyeCompassDirective
- Отображение ошибок формы
- Отправка данных регистрации на backend
- Поддержка переводов (i18n) для страницы регистрации
- Поддержка переключения темы для страницы регистрации
- Использование реактивной формы на странице регистрации

Это был мой первый самостоятельный компонент на Angular. Я изучала реактивные формы с нуля: разобралась, как работают `FormControl` и `FormGroup`, как подключить реактивные формы через импорт, как блокировать кнопку сабмита при невалидном состоянии формы.
 
Параллельно изучала библиотеку **Taiga UI** — несколько часов потратила на отладку глобальных стилей, пока не выяснилось, что нужно было просто подтянуть изменения из общей ветки.
 
Самостоятельно разобралась с переключением языков через **Transloco** — потратила на это около 6 часов с документацией и лекциями, и только потом узнала, что коллега уже написал инструкцию в вики.
 
Настроила общение с бэкендом. В процессе отладки несколько часов не могла понять, почему не приходят ответы от сервера — оказалось, в DevTools был включён фильтр консоли, из-за чего временно дебажила через `alert`. Позже переписала функцию отправки данных и заменила «заглушку» TypeScript через `!` на нормальную проверку:
 
```ts
if (!username || !email || !password) return;
```
 
Добавила отображение ошибок валидации (требования к паролю и т.д.), поправила компонент по комментариям из код-ревью после возвращения из поездки. Написала базовые тесты для компонента.
 
## Директива для глаз-компасов
- Ветка: `feat/FRONTEND_EyeCompass_CreateEyeCompassComponent`
- Реализация директивы Eye Compass
- Обработка движения мыши (mousemove) с привязкой к компоненту
- Корректная отписка от событий mousemove для оптимизации производительности  

Отдельная и достаточно нетривиальная задача. Вместе с коллегой около 9 часов пытались реализовать следящие за курсором глаза у котиков — сначала через компонент, потом через директиву. Ни то, ни другое не работало до четырёх утра. После сна прочитала документацию по директивам и реализовала корректно.
 
Позже, при помощи Антона — бывшего студента, который присоединился к нашему Discord-каналу — оптимизировала код директивы: убрала лишние шаги, теперь поиск нужных SVG-элементов идёт напрямую по DOM без лишних обходов.
 
## Игра Word Chain
Полностью самостоятельно разработала игру «Цепочка слов» на тему фронтенда. Написала всю логику с нуля:  
- Ветка: `feat/FRONTEND_ProgrammingCitiesGame_createCitiesGameForDev`
- Реализация страницы Cities Game 
- Управление состоянием сообщений (incoming / outgoing / system messages)
- Пошаговая логика диалога 
- Автоматический сценарий старта игры
- Логика инициализации и запуска игры
- Реализация restart механики (полный сброс состояния игры)
- Интеграция `tuiTextarea` для ввода пользовательского ответа
- Обработка отправки сообщения по клавише Enter
- Реализация игровой логики слов (валидация цепочки слов)
- Система подсказок
- Отображение аватара и username на игровой странице
- Реализация игрового таймера (start / stop / reset)
- Автоскролл чата к последнему сообщению (scroll-to-bottom)
- Route Guard для защиты игрового маршрута
- Настройка routing и логики входа в игровой режим
- Поддержка переводов (i18n) для страницы игры
- Поддержка переключения темы для страницы игры  
Полностью самостоятельно разработала игру «Цепочка слов» на тему фронтенда. Написала всю логику с нуля:
 
- кот проверяет, является ли введённое слово реальным фронтенд-термином
- реагирует на разные ситуации заготовленными фразами: неправильная буква, слово не по теме, слово уже использовалось, слова у кота закончились
- реализован чат с логикой отображения сообщений
- подготовлен словарь фронтенд-терминов
 
Попутно разобралась с жизненным циклом компонентов в Angular (`ngOnInit`), сигналами, таймерами и корректной их очисткой при уничтожении компонента.
 
## Дизайн и иллюстрации
 
С самого начала проекта занималась дизайном в **Figma**: отрисовала главную страницу, игровое поле, страницы входа и регистрации. В процессе разработки, правда, все поменяли. Отрисовала набор котиков для сайта с разными эмоциями и выражениями лица.
 
## Что было сложнее всего
 
Основная сложность — стартовать с нуля на незнакомом стеке при том, что я еще TypeScript очень похо знала. **Angular, Taiga UI, Transloco, реактивные формы, директивы, сигналы** — всё это я изучала прямо в ходе разработки. Много времени уходило не на написание кода, а на поиск нужного инструмента или понимание того, почему что-то не работает. Но каждый такой момент в итоге становился чем-то, что действительно запомнилось.
