## Описание моей работы

Основные компоненты, которые я разрабатывала:

- `User Profile`: спроектировала страницу профиля, собрала sidebar, блок recent activity, account settings, форму смены пароля, работу со статистикой, смену аватара и состояния загрузки.
- `user-store` и `user-service`: вынесла загрузку и обновление пользователя в отдельный store на `signals` и отдельный API-layer для запросов.
- `loader` и `route-loader`: сделала переиспользуемый loader в двух режимах (`page` и `inline`) и сервис для отображения глобального loader при lazy loading роутов.
- `Event Loop Game`: реализовала игру, включая загрузку задач, генерацию сессии по уровням сложности, drag-and-drop перестановку ответов, пошаговую проверку, подсказки, жизни, прогресс, финальные модалки и обновление статистики пользователя.

С какими инструментами и технологиями работала:

- `Angular`
- `Taiga UI`
- `Transloco`
- `RxJS`
- `signals`
- `Reactive Forms`
- `Angular CDK DragDrop`
- `ngx-highlightjs`
- `Cloudinary`
- работа с `HTTP` и backend API

Что было сложным:

- я пришла в проект без опыта работы с `Angular`, поэтому параллельно приходилось учить сам фреймворк и сразу применять его в реальном командном проекте;
- в `user-profile` пришлось разбираться с `signals`, формами, модалками, валидацией, загрузкой пользователя и синхронизацией UI после обновления данных;
- при смене аватара пришлось чинить интеграцию с `Cloudinary` и баг с `CORS`, который оказался связан с нашим interceptor;
- в игре самым сложным было удерживать корректное состояние во время асинхронной проверки ответов: блокировать повторные действия, обрабатывать выход из игры и не допускать конфликтов между попапами и состояниями.

## 2 личных Feature Component

### 1. User Profile

Это мой первый самостоятельный компонент. В нем я реализовала:

- структуру страницы и разбиение на подкомпоненты;
- работу `user-store` и `user-service`;
- формы редактирования данных и смены пароля;
- валидацию форм и отображение ошибок для пользователя;
- подтверждение изменения данных через модалку с вводом пароля;
- смену аватара через `Cloudinary`;
- показ статистики и recent activity;
- сортировку recent activity по дате последней активности;
- состояния загрузки для страницы, обновления данных и смены аватара;
- локализацию дат и текстов;
- условный рендеринг настроек для пользователей с GitHub-авторизацией.

### 2. Event Loop Game

Это мой второй большой компонент, который я сделала сама. В нем я реализовала:

- стартовый экран с описанием;
- код задачи с подсветкой синтаксиса;
- создание игровой сессии с подбором и перемешиванием задач по уровням сложности;
- drag-and-drop перестановку ответов;
- пошаговую проверку ответов;
- систему жизней и индикатор прогресса по заданиям;
- обработку победы, поражения, подсказок и выхода из игры;
- блокировку действий во время проверки, чтобы избежать повторных кликов и конфликтов состояний;
- анимированного котика-индикатор с реакцией на состояние игры и мурчанием;
- обновление статистики пользователя.
