## Account Settings

Ура. Наконец закончила с **Account Settings** для `UserProfile`. Настроила валидацию:

- при смене пароля проверяется, что новый пароль должен отличаться от старого
- проверяется, что поля для нового пароля и его подтверждения совпадают

Настроила появление модалки при смене `username` и `email` пользователя для подтверждения этой операции паролем.

### Проблема с модалкой

Я использую реализацию **Taiga UI**. Там модалку можно закрыть:

- по крестику
- по `Esc`

Я также добавила кнопку **Cancel**.

Сейчас настроен автофокус на инпуте с паролем при открытии модалки.
Если инпут невалидный — показывается ошибка.

Проблема:

- при нажатии на кнопку **Cancel** срабатывало событие с ошибкой потому что инпут `required` и не может быть пустым
- это происходит, если просто открыть модалку и закрыть её через **Cancel**

С другими способами закрытия проблем не было. По кнопке было видно, как на секунду появляется ошибка.

### Решение

Решение оказалось довольно простым — заменить:

```ts
click → pointerdown
```

После этого ошибка исчезла.

Также нужно было как-то вклиниться в дефолтные события закрытия модалки и добавить свою логику с очищением формы с паролем. В итоге оказалось, что это можно сделать через **effect**.

## Loader

На этой неделе нужно, чтобы был loader:

- при загрузке страницы
- и пока обрабатываются запросы

Разработать этот компонент выпало мне. Есть несколько вариантов, как это сделать.

### Минимальная реализация

Создать переиспользуемый компонент и вручную вставлять его в нужных местах. С запросами понятно — есть `loading`, по которому можно показывать компонент. А вот со страницами сложнее.

На моей странице сейчас загружается пользователь, так что есть флаг, по которому это можно добавить. Но есть и другие страницы, где это работает иначе.

Полезла искать информацию к чату. Оказалось, что можно сделать через `HttpInterceptor`. Это:

- круто
- переиспользуемо

Но пока хочется минимальной реализации. Так что вопрос пока ещё остаётся открытым.
