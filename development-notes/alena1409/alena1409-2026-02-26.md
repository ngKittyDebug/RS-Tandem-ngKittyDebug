### Cтруктуру данных для игры

Продумываю структуру данных для игры, которая будет храниться на бэкенде с поддержкой двух языков (ru / en):

```js
export interface CategoryPayload {
  id: string;
  category: {
    ru: string;
    en: string;
  };
  words: WordPayload[];
}

export interface WordPayload {
  id: string;
  word: string;
  questions: QuestionPayload[];
}

export interface QuestionPayload {
  id: string;
  question: {
    ru: string;
    en: string;
  };
  answer: {
    ru: string;
    en: string;
  };
  keywords: {
    ru: string[];
    en: string[];
  };
}
```

На текущий момент остаётся открытым вопрос загрузки данных из базы:

  - загружать ли сразу все данные на двух языках,
  - либо подгружать только выбранный пользователем язык.

От этого зависит:

- логика формирования ответа API,
- структура запроса,
- объём передаваемых данных,
- стратегия кеширования.

Пока нет окончательной ясности, какая архитектура будет более оптимальной.

Было очень интересно поработать с этим функционалом.

Однако сейчас реализация полностью находится на фронтенде, что архитектурно неверно.

Необходимо продумать:

- перенос логики на бэкенд,
