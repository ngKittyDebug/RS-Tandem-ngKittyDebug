#### AuthService

Написала по 2 теста на каждый метод: `login`, `register`, `refresh` — и 1 тест на `logout`.

Каждый метод проверяется в двух сценариях: успешный ответ сервера и ошибка.

Инструменты которые использовала:
- `provideHttpClientTesting()` — чтобы не отправлять реальные запросы на сервер
- `HttpTestingController` — чтобы перехватывать запросы и возвращать нужный ответ вместо сервера
- `req.flush()` — симулирует успешный ответ
- `req.flush(null, { status: 401 })` — симулирует ошибку

Паттерн каждого теста: **вызов метода → перехват запроса → ответ → проверка токена**.

Проверяла: сохраняется ли токен при успехе, обнуляется ли при ошибке, и корректно ли работает `isLoggedIn`.

#### Guards

Написала по 2 теста на каждый гард: `authGuard` и `guestGuard`.

Вместо реального `AuthService` использовала фейковый объект:
```typescript
{ provide: AuthService, useValue: { isLoggedIn: () => true } }
```

Это позволяет контролировать возвращаемое значение `isLoggedIn` без HTTP запросов.
