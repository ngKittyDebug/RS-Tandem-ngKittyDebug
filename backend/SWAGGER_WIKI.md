## Что такое Swagger UI?

**Swagger UI** — это интерактивный интерфейс для документирования и тестирования REST API. Он позволяет:

- 📖 **Изучать документацию** — все эндпоинты с описаниями, параметрами и примерами
- 🧪 **Тестировать API** — отправлять запросы прямо из браузера без Postman/cURL
- 🔐 **Управлять авторизацией** — сохранять JWT токен для защищённых эндпоинтов
- 📥 **Экспортировать спецификацию** — скачать OpenAPI документ в формате YAML

## Интерфейс Swagger UI

### Основные элементы

| Элемент              | Описание                                                                        |
| -------------------- | ------------------------------------------------------------------------------- |
| **Название API**     | Вверху страницы — название проекта (MeowVault)                                  |
| **Секции**           | Эндпоинты сгруппированы по модулям: `Auth`, `User`, `Key Storage`, `Merge Game` |
| **Методы**           | Цветные метки: 🟢 GET, 🔵 POST, 🟡 PATCH, 🔴 DELETE                             |
| **Кнопка Authorize** | 🔑 Вверху справа — для ввода JWT токена                                         |
| **Try it out**       | Кнопка для активации формы запроса                                              |

### Легенда значков

| Значок | Описание                                                     |
| ------ | ------------------------------------------------------------ |
| 🔓     | **Публичный** — не требует авторизации (декоратор `@Public`) |
| 🔐     | **JWT** — требует JWT токен (Bearer Auth)                    |
| 👑     | **Admin** — требует JWT токен + роль **admin**               |
| 🟢     | **GET** — получение данных                                   |
| 🔵     | **POST** — создание данных                                   |
| 🟡     | **PATCH** — обновление данных                                |
| 🔴     | **DELETE** — удаление данных                                 |

## Как использовать Swagger UI для тестирования

### Пошаговая инструкция

```
1. Раскройте эндпоинт (клик на названии)
         │
         ▼
2. Нажмите кнопку "Try it out"
         │
         ▼
3. Введите данные в форму запроса
         │
         ▼
4. Нажмите кнопку "Execute"
         │
         ▼
5. Просмотрите ответ (статус, заголовки, тело)
```

### Пример: Регистрация пользователя

1. Найдите секцию **Auth** → `POST /auth/register`
2. Нажмите **Try it out**
3. Вставьте JSON в поле **Request body**:
   ```json
   {
     "email": "test@example.com",
     "username": "test_user",
     "password": "TestPass123!"
   }
   ```
4. Нажмите **Execute**
5. В разделе **Responses** увидите:
   - Status: `201`
   - Response body: `{ "accessToken": "..." }`
6. Скопируйте токен для авторизации

## Авторизация в Swagger UI

### Как получить токен

1. Выполните запрос `POST /auth/login` или `POST /auth/register`
2. Скопируйте значение `accessToken` из ответа
3. Используйте его для авторизации

### Как авторизоваться

1. Нажмите кнопку **Authorize** 🔑 (вверху справа)
2. В поле введите токен в формате: `Bearer <ваш_токен>`
3. Нажмите **Authorize**, затем **Close**

> ✅ После авторизации токен сохраняется в браузере и используется для всех защищённых запросов.

## Обзор модулей

### 🔓 Auth Module

**Базовый путь:** `/auth`  
**Особенность:** Все эндпоинты помечены `@Public()` — не требуют JWT

| Метод   | Эндпоинт         | Описание                 | Доступ       |
| ------- | ---------------- | ------------------------ | ------------ |
| 🔵 POST | `/auth/register` | Регистрация пользователя | 🔓 Публичный |
| 🔵 POST | `/auth/login`    | Вход в систему           | 🔓 Публичный |
| 🔵 POST | `/auth/refresh`  | Обновление токенов       | 🔓 Публичный |
| 🔵 POST | `/auth/logout`   | Выход из системы         | 🔓 Публичный |

---

### 🔐 User Module

**Базовый путь:** `/user`  
**Особенность:** Все эндпоинты требуют JWT (глобальный `JwtAuthGuard`)

| Метод     | Эндпоинт         | Описание                      | Доступ |
| --------- | ---------------- | ----------------------------- | ------ |
| 🟢 GET    | `/user`          | Получение данных пользователя | 🔐 JWT |
| 🟡 PATCH  | `/user/profile`  | Обновление профиля            | 🔐 JWT |
| 🟡 PATCH  | `/user/avatar`   | Обновление аватара            | 🔐 JWT |
| 🟡 PATCH  | `/user/password` | Смена пароля                  | 🔐 JWT |
| 🔴 DELETE | `/user`          | Удаление аккаунта             | 🔐 JWT |

---

### 🗄️ Key Storage Module

**Базовый путь:** `/key-storage`  
**Особенность:** GET — требуют JWT, POST/DELETE — требуют JWT + admin

| Метод     | Эндпоинт              | Описание                | Доступ   |
| --------- | --------------------- | ----------------------- | -------- |
| 🔵 POST   | `/key-storage`        | Создание записи         | 👑 Admin |
| 🟢 GET    | `/key-storage/all`    | Получение всех записей  | 🔐 JWT   |
| 🟢 GET    | `/key-storage/params` | Поиск по ключу/значению | 🔐 JWT   |
| 🔴 DELETE | `/key-storage`        | Удаление записи         | 👑 Admin |

> **GET** — требуют JWT, **POST/DELETE** — требуют JWT + роль **admin**.

## Поддержка

При возникновении проблем:

1. Проверьте, что сервер запущен (`npm run start:dev`)
2. Убедитесь, что порт 3000 не занят другими приложениями
3. Проверьте логи сервера на наличие ошибок
4. Убедитесь, что JWT токен действителен и не истёк
